# AGLDispatchBot

Telegram-бот для логистических компаний **Arrow Green Logistics (AGL)** и **BALTCRAFT**. 
Позволяет менеджерам быстро и удобно оформлять заявки на грузоперевозки и публиковать их в общий канал.

## Возможности
- Оформление заявок пошагово (включает выбор компании, маршрута, груза, дат, транспорта, условий).
- Быстрый пропуск необязательных полей с помощью "-".
- Пометка "🔴 СРОЧНАЯ ЗАЯВКА", если в сроках указано слово "срочно".
- Редактирование созданных заявок с автоматическим уведомлением об изменениях в общий канал.
- Просмотр карточек и списка последних заявок.
- Все временные метки автоматически приводятся к Ташкентскому времени (UZT).

## Требования
- Python 3.9+
- `python-telegram-bot` >= 20.0
- `python-dotenv`
- `pytz`

## Локальный запуск

1. Склонируйте или скачайте репозиторий.
2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   python -m venv venv
   source venv/bin/activate       # Для Linux/MacOS
   venv\Scripts\activate          # Для Windows
   pip install -r requirements.txt
   ```
3. Создайте файл `.env` на основе `.env.example` и укажите ваши данные:
   ```env
   BOT_TOKEN=ваш_токен_бота_от_BotFather
   CHANNEL_ID=-100...
   ```
4. Запустите бота:
   ```bash
   python bot.py
   ```

## Инструкция по деплою на Railway.app

Railway.app — это удобная платформа для деплоя ботов прямо из GitHub репозитория.

### Шаг 1: Подготовка
1. Загрузите код вашего бота в новый репозиторий на **GitHub**. (включая файлы `bot.py`, `requirements.txt`).
2. **Внимание:** Файлы `requests.json` и `.env` загружать не нужно (добавьте их в `.gitignore`).

### Шаг 2: Создание проекта в Railway
1. Зарегистрируйтесь или войдите на [Railway.app](https://railway.app/).
2. Нажмите **New Project** -> **Deploy from GitHub repo**.
3. Выберите репозиторий с вашим ботом.
4. Нажмите **Deploy Now**.
5. Railway автоматически найдет `requirements.txt` и соберет проект.

### Шаг 3: Настройка переменных окружения
1. Перейдите в настройки вашего проекта в Railway, раздел **Variables**.
2. Нажмите **New Variable** и добавьте:
   - `BOT_TOKEN` со значением вашего токена.
   - `CHANNEL_ID` со значением ID вашего Telegram-канала.
3. Проект будет автоматически перезагружен с новыми переменными.

### Шаг 4: Хранение данных (Важно!)
По умолчанию файловая система Railway является **эфимерной**, то есть файл `requests.json` может быть удален при перезапуске сервера (например, при обновлении переменных).
Для постоянного хранения вы можете:
1. Использовать Railway Volume (диск, который прикрепляется к сервису). Для этого перейдите в настройки сервиса -> **Volumes** и создайте новый том.
2. Примонтируйте его по пути (например, `/app/data`).
3. В коде `bot.py` поменяйте `DATA_FILE = "requests.json"` на `DATA_FILE = "/app/data/requests.json"`.

Либо перенести базу в PostgreSQL или MongoDB (в будущем), когда объем заявок станет значительным.
